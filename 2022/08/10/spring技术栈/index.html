spring技术栈描述

  • 1.Reactor Core/Reactor Project
    反应式编程，开发webflux项目

  • 2.Spring Batch
    批处理架构，执行job的调度任务和与job交互，一个job可以分为多个step进行处理数据

  • 3.Spring Data JPA
    持久化框架

  • 4.Spring Security
    安全框架，为 Java 应用程序提供身份验证和授权的框架

  • 5.Spring Boot
    用最小的配置轻松创建spring程序

  • 6.Spring Cloud Task
    允许用户将几乎任何 Spring Boot 应用程序作为短期任务执行
    Spring Cloud Task可以在平台环境中执行短生命周期的应用，并记录这些应用的结果，
    经由通过消息通知的机制来集成各个tasks。通过这特性就可以实现让短生命周期的应用像长生命周期的应用一样组成微服务架构。

  • 7.Spring AMQP
    将核心 Spring 概念应用于基于 AMQP 的消息传递解决方案的开发

  • 8.Spring Cloud Stream
    为了解决系统中不同中间件的适配问题，出现了cloud stream，采用适配绑定的方式，自动给不同的MQ之间进行切换。
    目前仅仅支持RabbitMQ和Kafka

  • 9.Spring Data REST
    Spring Data REST把我们需要编写的大量REST模版接口做了自动化实现
    只需编写实体类和dao层接口就会自动隐藏式生成相关的restful增删改查和分页查询等也可以自定义接口

  • 10.Spring Cloud Data Flow
    基于微服务的，专门用于流式和批式数据处理的框架（相当于Spring Batch 和 Spring Cloud Task的控制台）

  • 11.Spring HATEOAS
    创建遵循 HATEOAS 原则的 REST 表示，在资源的表达中包含了链接信息。客户端可以根据链接来发现可以执行的动作

  • 12 Spring LDAP
    简化 Java 中的 LDAP 编程的库

  • 13 Spring XD
    Spring XD 是一个统一的，分布式，可扩展的系统用于数据摄取，实时分析，批量处理和数据导出。该项目的目标是简化大数据应用的开发

  • 14 Spring Cloud Consul
    Consul是 HashiCorp 公司推出的开源工具，用于实现分布式系统的服务发现与配置

  • 15 Spring Session
    解决分布式系统session共享问题

  • 16 Spring Statemachine
    供应用程序开发人员在 Spring 应用程序中使用状态机概念
