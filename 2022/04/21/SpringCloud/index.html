微服务技术对比

Dubbo SpringCloud SpringCloudAlibaba
注册中心 zookeeper/redis Eureka Nacos/Eureka
服务远程调用 Dubbo协议 Feign(http协议) Dubbo/Feign
配置中心 无 SpringCloudConfig SpringCloudConfig/Nacos
服务网关 无 SpringCloudGateway/zuul SpringCloudGateway/zuul
服务监控和保护 dubbo-admin Hystrix Sentinel

Eureka

SpringCloud使用http协议发起远程调用请求
比如 restTemplate.getForObject(“localhost:8081/user/“+user.getId(), User.class)

使用Eureka提供服务的注册和发现，使用服务名调用
restTemplate.getForObject(“UserService/user”+user.getId(), User.class)

Ribbon

Ribbon负载均衡流程

orderService  --1.发起请求 http://userservice/user/1-->   4.轮询到某台机 userservice 8001
                        |                   |    
                        |                   |
                      2.拉取userservice     3.返回服务列表
                        |                   |
                        |                   |
                            eureka-service

Ribbon使用IRule进行负载均衡规则的选择
可以通过bean或修改配置文件修改负载均衡策略

Ribbon是懒加载，第一次访问才会创建loadbalanceClient,请求时间很长，
配置饥饿加载可以在项目启动时创建，降低第一次访问的时间（服务预热）

Nacos

Nacos集群服务管理

服务的注册与发现（CSF控制台）
nacos会展示服务对应的实例及实例地址（关注实例的状态可以给实例设置权重），
csf主要展示服务的配置信息（路由策略，负载类型，降级策略，超时设定）

服务分级存储
以机房划分集群，比如：杭州集群、上海集群（CSF的租户概念）

nacos使用namespace做环境隔离，不同环境无法相互访问

Nacos和Eureka区别
1.nacos有临时实例和非临时实例，临时实例心跳监测（失效会被直接剔除），非临时实例nacos主动询问（会一直保持实例）
2.nacos主动推送变更消息

Nacos统一配置管理（K8S统一配置）

统一配置管理，可以进行热更新配置

Feign

使用feign代替resttemplate，声明式远程调用（http客户端）

Feign底层客户端实现：
URLConnection 默认不支持连接池
Apache HttpClient
OKHttp

Feign性能优化
1.使用带连接池的替换URLConnection
2.修改日志级别为BASIC和NONE

Feign的最佳实现：
1.让controller和feignclient继承相同接口
2.Feign抽取调用模块，提供给所有消费者使用

GateWay

网关功能：
1.身份认证和权限校验
2.服务路由和负载均衡
3.请求限流

zuul基于servlet，属于阻塞式编程，gateway属于响应式编程，具有更好的性能

路由断言工厂：（满足断言才会路由）
path 请求路径必须符合指定规则
before 在某个时间点前请求
after 在某个时间点后请求

路由过滤器

对进入网关的请求和微服务的响应进行过滤

针对部分路由的过滤器
针对所有路由的过滤器
处理一切进入网关的请求和响应（全局过滤器）逻辑自定义

网关CORS（浏览器询问服务器能否跨域）跨域跨域配置

www.taobao.com www.taobao.org miaosha.taobao 域名不同的跨域请求
