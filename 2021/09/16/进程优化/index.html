托收算费优化

2021-09-16 调优 737 字
进程优化

场景：新上线常州分公司的托收任务（20W数据） 算费过慢 目标（每分钟处理 5000条数据）

分析：
	一条数据共耗时：2719ms  查询托收关系表
	1.  92ms  select * from AM_ENTRUST_RELATION where 1 = 1  and state = ? and ACCT_ID = ? 
	--加索引，gj2环境已经加了
	2.  824ms   EntrustFunc.qryAcctAllStopServ ,判断是否为主动停机用户
	--建议查询计费用户状态表
	3. 214ms 从注册中心获取远程服务地址:OC.BC.person.IQryCustIdSV.checkCustExistsByType->http://10.97.114.108:10001/service
	--建议查询计费用户状态表
	4. 542ms 从注册中心获取远程服务地址:OC.person.IQueryScaInfoSV.queryUserCustAcctInfoByAccId->http://10.97.114.108:10001/service
	--建议查询计费用户状态表
	5. 412ms select b.service_id,a.object_id,min(a.PRODUCT_STATUS) status_min,max(a.PRODUCT_STATUS) status_max,min(a.VALID_DATE) valid_date,max(a.EXPIRE_DATE) expire_date from CD.CO_PROD_STS_0523_1 a left join CD.CO_PROD_0523_1 b on a.PRODUCT_ID=b.PRODUCT_ID and b.service_id is not null where a.OBJECT_TYPE= 0 and a.PRODUCT_STATUS !=0 and a.EXPIRE_DATE>=sysdate and a.OBJECT_ID in (108727164,108727147,102349814) group by a.object_id,b.SERVICE_ID 
	--优化索引，gj2环境已经加了--50ms左右
	6. 606ms 本次调用服务asset_IPaymentCSV_queryOweFee总耗时:606ms
	
	30分钟大概处理4000条数据
第一次优化：
	1.优化表，表加索引
	2.减少去三户信息的调用  查表替代
	3.优化查欠      查欠不查询三户信息，托收不需要获取客户的三户信息
	
结果：
	目前效率 1分钟500条     500 * 60 = 3W条
	大部分使用CPU（5-6）  服务器：32核
	提升线程数也不能提供CPU的使用

分析：
	CPU占用上不去，进程太少
	多的分公司有20W数据，优化拆分逻辑，对账户进行取模分发
	（分发数据前修改查询方式）
	第二次优化：
		添加通道，对MOD(A.CUST_CODE,10000) 0-1999 2000-3999 4000-5999 6000-7999 8000-9999 在分5个进程处理
		
		
	结果：
		每分钟大概3500条
		
	
	分析：
[2021-09-21 14:55:33,715] [processor-6] (ClientStub.java:119)  INFO  - 本次调用服务asset_ICycleCSV_querySysPreCycle总耗时:394ms
[2021-09-21 14:55:34,124] [processor-6] (ClientStub.java:119)  INFO  - 本次调用服务asset_ICycleCSV_querySysCurCycle总耗时:342ms
[2021-09-21 14:55:34,377] [processor-6] (ClientStub.java:119)  INFO  - 本次调用服务asset_ICycleCSV_querySysMaxCycle总耗时:250ms
[2021-09-21 14:55:34,966] [processor-6] (ClientStub.java:119)  INFO  - 本次调用服务acomp_ICommParamCSV_getList总耗时:570ms
[2021-09-21 14:55:35,406] [processor-6] (ClientStub.java:119)  INFO  - 本次调用服务acomp_ICommParamCSV_getList总耗时:435ms
[2021-09-21 14:55:36,051] [processor-6] (ClientStub.java:119)  INFO  - 本次调用服务asset_IEntrustDoCSV_doEntrustErr总耗时:618ms
		
		发现多次去调用服务查询账期（查询虚销时需要查询账期）
		实际获取的都是系统账期（当前没有分库），只需要查询一次就可以了
第三次优化：
	进程启动时查询账期数据放入缓存，
	查欠修改支持外部传入账期，
	账期只有第一次需要查询，之后直接获取即可
	

结果：
	2台主机，30个进程，10个线程，5000条/分钟--7000条/分钟
