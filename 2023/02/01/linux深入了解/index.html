linux深入了解

2023-02-01 linux 2.3k 字
学习

常用指令

文件目录

  • cat 查看文件内容
  • more 文本过滤器
  • less 与more相似，大型文件具有较高效率
  • **>**覆盖 **>>**追加
  • ln 软链接 target linkname
  • history 查看已经执行的历史命令 history 10 最近10个指令 !10 执行第10条指令

时间日期

  • date 展示时间 date “+%Y-%m-%d %H:%M:%S” 定义格式
  • date -s “2022-02-04 11:19:20” 修改系统时间
  • cal 查看日历 [2023] 展示整年日历

查找

  • find [搜索范围] [选项] 从指定目录向下递归搜索
    • -name 按照文件名查找
    • -user 查找属于指定用户的文件
    • -size 指定文件大小查找 (+n大于 -n小于 n等于)
  • locate 可以快速定位文件路径，第一次运行必须使用updatedb创建locate数据库
  • grep [选项] 查找内容 源文件, 过滤查询

压缩和解压

  • gzip/gunzip 压缩成*.gz文件

  • zip/unzip 压缩文件或文件夹成*.zip

  • tar 打包指令，打包文件是.tar.gz

    • -c 产生.tar打包文件
    • -v 显示详细信息
    • -f 指定压缩后文件名
    • -z 打包同时压缩
    • -x 解包.tar文件

    压缩多个文件，将a.txt和b压缩成ab.tar.gz

    tar -zcvf ab.tar.gz a.txt b

    解压ab.tar.gz

    tar -zxvf ab.tar.gz

组管理和权限管理

  • chown 用户名 文件名 修改文件所有者

  • chgrp 组名 文件名 修改文件所在组

  • usermod -g 新组名 用户名 改变用户所在组

  • chmod 修改权限

    • +/-/=变更权限

      u所有者 g所有组 o其他人 a所有人

      chmod u=rwx,g=rx,o=x 文件/目录名

      chmod o+w 文件/目录名 给其他人赋予写的权限

      chmod a-x 文件/目录名 所有人去除执行的权限

    • 通过数字变更权限

      r=4,w=2,x=1（1x,2w,3wx,4r,5rx,6rw,7rwx）

      chmod u=rwx,g=rx,o=x 文件/目录名

      相当于 chmod 751 文件/目录名

权限说明
drwxr-xr-x
0123456789
1.第0位确认文件类型(d,-,l,c,b)
	l链接
	d目录
	c字符设备，鼠标，键盘
	b块设备，硬盘
	-文件
2.第1-3确认文件所有者拥有该文件的权限 --User
3.第4-6确认文件所属组拥有该文件的权限 --Group
4.第7-9确认其他用户拥有该文件的权限 --Other

rwx作用到文件
1.[r]可读
2.[w]可写，不代表可删，删除文件需要对所在目录有写的权限
3.[x]可执行
rwx作用到目录
1.[r]可读，可以列出目录下的文件
2.[w]可写，对目录内创建，删除，重命名
3.[x]可执行，可以进入该目录

定时任务调度

  • crontab [-e]编辑crontab定时任务 [-l]查询crontab 任务 [-r]删除当前用户所有的crontab 任务
cron表达式
第一个*  一小时中第几分钟  0-59
第二个*  一天中第几小时    0-23
第三个*  一个月中第几天    1-31
第四个*  一年中第几月      1-12
第五个*  一周中星期几      0-7（0/7都代表星期日）
* 代表任何时间
，代表不连续的时间
- 代表连续的时间
*/n  代表每隔多久执行一次

  • at [选项] [时间] 一次性定时任务

    1.接受当天的hh:mm（小时：分钟），如果时间过去，明天执行
    2.midenight,non,teatime等模糊时间
    3.12小时制 12pm
    4.命令执行具体时间 
    5.相对计时 2天后下午5点  
    	at 5pm + 2 days
    6.直接使用today/tomorrow   明天下午5点
    	at 5pm tomorrow

磁盘分区

  • lsblk [-f] 查看设备挂载情况

  • fdisk /dev/sdb 分区命令

  • mkfs -t ext4 /dev/sdb1 格式化磁盘

  • mount /dev/sdb1 /newdisk 挂载文件系统

  • df -h 磁盘情况查询

  • du 查询指定目录磁盘占用情况

    • -s 指定目录
    • -h 带计量单位
    • -a 带上文件
    • -c 汇总
    • –max-depth=1 子目录深度
    查询/opt下文件的个数(过滤出以“-”开头的就是文件)
    ls -l /opt | grep "^-" | wc -l
    查询/opt文件个数包括子目录
    ls -lR /opt | grep "^-" | wc -l
    树型展示目录结构
    tree /opt

进程管理

  • ps 查看系统中的进程
    • -a 显示当前终端所有进程信息
    • -u 以用户格式显示进程信息
    • -x 显示后台进程运行的参数
    • -e显示所有进程 -f 全格式。显示所有进程，可以查看进程的父进程
  • kill [选项] 进程号 终止进程 -9 强制进程终止
  • killall 进程名 通过进程名杀死进程，支持通配，也会杀死子进程
  • pstree 更加直观展示进程
    • -p 展示pid
    • -u 显示进程所属用户

服务管理

  • service [start|stop|restart|reload|status] 服务名
  • systemctl get-default 获取运行级别
  • systemctl [start|stop|restart|reload|status] 服务名
    • 管理的服务在/usr/lib/systemd/system查看
    • systemctl list-unit-filessystemctl 查看服务自启动状态
    • systemctl enable(disable) 开启或关闭服务自启动
    • systemctl is-enabled 服务名 查看服务自启动状态
  • firewall 防火墙
    • 打开端口 firewall-cmd –permanent –add-port=端口号/协议
    • 关闭端口 firewall-cmd –permanent –remove-port=端口号/协议
    • 重新载入才能生效 firewall-cmd –reload
    • 查询端口是否开放 firewall-cmd –query-port=端口号/协议
  • top 动态监控 [-p 监控指定进程ID] [-d 刷新间隔]
    • P 以CPU使用率排序
    • M 以内存使用率排序
    • u 监控指定用户
    • k 终止指定的进程
  • netstat 监控网络状态 [-p 展示那个进程在调用] [-an 按一定顺序排列]

软件包管理

  • rpm

    • 查询已安装的rpm rpm -qa | grep xx
    • firefox-60.2.2-1.el7.centos.x86_64
      • firefox 名称
      • 60.2.2-1 版本号
      • el7.centos.x86_64 适用操作系统 noarch适用通用版本
    • rpm -q 软件包 查询软件包是否安装
    • rpm -e 软件包 卸载
    • rpm -ivh RPM包全路径名称

  • yum 能够从指定服务器自动下载RPM包并安装

    • yum list | grep xx软件列表 查询yum服务器是否有需要安装的软件
    • yum install xxx 下载安装

  • apt

    • /etc/apt/sources.list 服务器地址

    • apt软件操作

      sudo apt-get update 更新源
      sudo apt-get install package  安装包
      sudo apt-get remove package 删除包
      sudo apt-cache show package 获取包相关信息，如说明/大小/版本等
      sudo apt-get source package 下载该包的源代码

shell

  • 位置参数

    • $n $0命令本身，$1-$9代表第一到第九个参数，十以上${10}
    • $* 代表命令行中所有参数整体对待
    • s@ 命令行中所有参数分别对待
    • s# 命令行中参数个数

  • 预定义变量

    • $$ 当前进程的进程号
    • $! 后台运行的最后一个进程的进程号
    • $? 最后一次执行命令返回状态，返回0证明上一个命令正确执行

  • 运算符

    • $((运算符)) 或 $[运算符] expr m + n

  • 条件判断

    • 判断语句 [ condition ] (condition前后要有空格)
    • = 字符串比较
    • 2个整数比较
      • -lt 小于
      • le 小于等于
      • eq 等于
      • gt 大于
      • ge 大于等于
      • ne 不等于
    • 按照文件权限进行判断
      • -r 有读的权限
      • -w 有写的权限
      • -x 有执行的权限
    • 按照文件类型进程判断
      • -f 文件存在并且是一个常规文件
      • -e 文件存在
      • -d 文件存在并是一个目录

  • case

    #!/bin/bash
    case $1 in
    "1")
            echo "monday"
    ;;
    "2")
            echo "tusday"
    ;;
    *)
            echo "other"
    ;;
    esac

  • for循环

    • 基本语法1

      for 变量 in 值1 值2 值3

      do

      程序

      done

    • 基本语法2

      for (( 初始值;循环控制条件;变量变化 ))

      do

      程序

      done

  • while循环

    while [ condition ]

    do

    程序

    done

  • read 读取控制台输入

    -p 指定读取值时的提示符

    -t 指定读取时等待的时间

  • 函数

    • 系统函数 basename

      返回完整路径最后/的部分，常用于获取文件名

      dirname 返回完整路径最后/前面的部分，用于返回路径

    • 自定义函数

      #!/bin/bash
      function getSum()
      {
              SUM=$[$num1+$num2]
              echo "sum=$SUM"
      }
      read -p "请输入第一个数num1=" num1
      read -p "请输入第二个数num2=" num2
      getSum $num1 $num2

日志

  • 查看内存日志

    journalctl -n 3 查看最新3条

    journalctl –since 19:00 –until 19:10:10 查看时间区间

    journalctl -p err 报错日志

    journalctl -o verbose 日志详细内容

    journalctl _PID=1245 _COMM=sshd 查看包含这些参数的日志

备份与恢复

  • dump 对分区支持增量备份

    dump -0j -f /opt/etc.bak.bz2 /etc

  • restore 用来恢复已备份的文件

    restore [模式选项] [选项]

    四种模式不能混用

    • -C 对比模式，将备份文件与已存在文件相互对比
    • -i 使用交互模式，进行还原时，restore 指令依序询问用户
    • -r 还原模式：如果有增量备份，需要把增量文件也按照顺序进行恢复
    • -t 查看模式，看备份文件有哪些文件

    选项

    • -f 从指定文件中读取备份数据进行还原
