RUST入门

1. 简介

  • rust是一种令人兴奋的新编程语音，它可以让每个人编写可靠且高效的软件。
  • 它可以用来替换C/C++，rust和它们具有相同的性能，但是很多常见的bug在编译时就可以被消灭。

2. 基础命令

# 编译执行
cargo run   
# 检查代码
cargo check
# 为发布构建
cargo build --release

3. 通用的编程概念

3.1 变量与可变性

shadowing

可以使用相同的名字声明新的变量，新的变量就会shadow(隐藏)之前声明的同名变量

3.2 数据类型：标量类型

标量类型代表一个单个的值，主要标量类型：

  • 整数类型 Signed i32 Unsigned u32
  • 浮点类型 f32 单精度 f64 双精度
  • 布尔类型 true false
  • 字符类型 char

3.3 数据类型：复合类型

复合类型可以将多个值放在一个类型里。rust提供两种基础的复合类型：元组（Tuple）、数组

Tuple

tuple可以将多个类型的多个值放在一个类型里；tuple长度固定，一旦声明无法改变

let tup: (i32, f64, u8) = (500, 6.4, 1);

数组

数组可以将多个值放在一个类型里；数组中每个元素类型必须i相同；数组的长度也是固定的。

数组的类型表达形式：[类型；长度]

let a: [i32;5]=[1,2,3,4,5];

3.4 函数

函数签名里，必须声明每个参数的类型。

在 ->符号后面声明函数返回值的类型，但不可以为返回值命名。

在rust里面，返回值就是函数里面最后一个表达式的值。

3.5 控制流

如果使用多于一个else if,那么最好使用match来重构代码。

因为if是一个表达式，所有可以将它放在let语句中等号的右边

3.6 控制流：循环

rust提供3种循环：loop,while,for

  • loop告诉rust反复执行一块代码，直到喊停
  • while循环每次执行循环体之前都判断一次条件

4. 所有权

4.1 stack vs heap

存储数据

  • 把值压到stack上不叫分配，因为指针是已知固定大小的，可以把指针存放在stack上（如果想要实际数据，必须使用指针来定位）
  • 把数据压到stack上比heap上分配快的多（操作系统不需要寻找新数据的空间，那个位置永远在stack顶端）
  • 在heap上分配空间需要更多工作（操作系统需要找到一个足够大的空间来存放数据，然后要做好记录方便下次分配）

所有权存在原因：

  • 跟踪代码的哪些部分正在使用heap的哪些数据
  • 最小化heap上重复数据量
  • 清理heap上未使用的数据以避免空间不足

4.2 所有权规则

  • 每个值都有一个变量，这个变量是该值的所有者
  • 每个值同时只能有一个所有者
  • 当所有者超出作用域时，该值将被删除

变量和数据交互方式：移动

    let s1 = String::from("hello");
    let s2 = s1;
    println!("{}", s1);
# value moved here
# move occurs because `s1` has type `String`, which does not implement the `Copy` trait

rust没有尝试复制被分配的内存

rust让s1失效。（当s1离开作用域时，rust不需要释放任何东西）

stack上的数据：复制

let x = 5;
let y = x;
println!("{},{}", x,y);

一个类型实现了Copy这个trait，那么旧的变量在赋值后任然可用

4.3 所有权与函数

在语义上将值传递给函数和把值赋给变量是类似：将值传递给函数将发生移动或复制

函数在返回值的过程同样会发生所有权的转移

4.4 引用与借用

fn main() {
    let s1 = String::from("Hello");
    let len = calculate_length(&s1);
    println!("The length of '{}' is {}.", s1, len);
}

fn calculate_length(s: &String) -> usize {
    s.len()
}

&符号表示引用：允许你引用某些值而不取得其所有权

不可以修改借用的值，和变量一样，引用默认也是不可变的

允许可变的引用，但会有限制：在特定作用域内，对某块数据，只能有一个可变引用；不可以同时拥有一个可变引用和一个不变的引用；多个不变引用是可以的

引用的规则

  • 在任何给定的时刻，只能满足下列条件之一：
    • 一个可变的引用
    • 任意数量不可变的引用
  • 引用必须一直有效

4.5 切片

字符串切片是指向字符串中一部分内容的引用 [开始索引..结束索引]

let s = String::from("Hello world");

let hello = &s[0..5];
let world = &s[6..11];

定义函数时使用字符串切片来代替字符串引用会使我们的API更加通用，且不会损失任何功能。

字符串字面值是切片 let s = “hello world”

// 该struct实例拥有其所有的数据

一旦struct的实例是可变的，那么实例中所有的字段都是可变的

tuple struct

tuple struct整体有个名，但里面的元素没有名

struct Color(i32,i32,i32);
let black = Color(0,0,0);

struct方法

#[derive(Debug)]
struct Rectangle {
    width : u32,
    length: u32,
}

impl Rectangle {
    // 方法第一个参数是self,表示方法被调用的struct实例
    fn area(&self) -> u32 {
        self.width * self.length
    }
    // 不把self作为第一个参数的函数，它们叫关联函数
    fn square(size: u32) -> Rectangle {
        Rectangle { width: size, length: size }
    }
}

fn main() {
    let sq = Rectangle::square(20);
    let rect = Rectangle {
        width: 30,
        length: 50,
    };
    println!("{}", rect.area());
    println!("{:#?}", rect);
}

关联函数

可以在impl块里定义不把self作为第一个参数的函数，它们叫关联函数。关联函数通常用于构造器 ::符号关联函数

6.3 控制流运算符 - match

#[derive(Debug)]
enum UsState {
    Alabama,
    Alaska,
}

enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter(UsState),
}

fn value_in_cents(coin: Coin) -> u8 {
    match coin {
        Coin::Penny => 1,
        Coin::Nickel => 5,
        Coin::Dime => 10,
        Coin::Quarter(state) => {
            println!("State quarter from {:?}!", state);
            25
        },
    }
}

fn main(){
    let c = Coin::Quarter(UsState::Alaska);
    println!("{}", value_in_cents(c));
}

match匹配必须穷举所有的可能，或者使用_通配符替代其余没列出的值

    let v = Some(0u8);
    match v {
        Some(3) => println!("three"),
        _ => (),
    }
    if let Some(3) = v {
        println!("three");
    }
// 只关心一种匹配而忽略其他匹配情况

push_str()：把一个字符串切片附加到String

let s1 = String::from("tic");

8.3 HashMap

在元素类型为tuple的vector上使用collect可以组建一个hashmap

let teams = vec![String::from("Blue"), String::from("Yellow")];
let intial_scores = vec![10,50];
let scores: HashMap<_,_>=teams.iter().zip(intial_scores.iter()).collect();

entry方法：检查指定的K是否对应一个V

Entry的or_insert()方法：如果K存在，返回一个对应V的可变引用；如果K不存在，将方法参数作为K的新值插入，返回这个值的可变引用

let mut scores = HashMap::new();
scores.insert(String::from("Blue"), 10);

scores.entry(String::from("Yellow")).or_insert(50);
scores.entry(String::from("Blue")).or_insert(50);

println!("{:?}", scores);

9. 错误处理

9.1 panic!

当panic!执行时，程序会打印错误信息，展开/清理调用栈，并退出程序

9.2 result

处理result的一种方式：match

let f = File::open("hello.txt");

let _f = match f {
    Ok(file) => file,
    Err(error) => match error.kind() {
        ErrorKind::NotFound => match File::create("hello.txt"){
            Ok(fc) => fc,
            Err(e) => panic!("Error creating file: {:?}", e),
        },
        other_error => panic!("Error opening file: {:?}", other_error),
    },
};

unwarp

unwarp是match表达式的一个快捷方法，如果Result结果是Ok,返回ok里面的值，如果result结果是err,调用panic!宏

expect

expect作用和unwarp类似，但可以自定义错误信息

传播错误

函数不处理错误，将错误返回给调用者

fn read_username_from_file() -> Result<String, std::io::Error>{
    let f = File::open("hello.txt");

    let mut f = match f {
        Ok(file) => file,
        Err(e) => return Err(e),
    };

    let mut s = String::new();
    match f.read_to_string(&mut s){
        Ok(_) => Ok(s),
        Err(e) => Err(e),
    }
}

***?***传播错误的一种快捷方式

如果result是ok，那ok中的值就是表达式的结果，然后继续执行，

如果result是err,err就是整个函数的返回值，就像使用了return

fn read_username_from_file() -> Result<String, std::io::Error>{
    // 使用？和链式调用简化代码
    let mut s = String::new();
    File::open("hello.txt")?.read_to_string(&mut s)?;
    Ok(s)
}

10. 泛型，Trait， 生命周期

10.1 泛型

泛型是具体类型或其他属性的抽象代替，编写模板，在编译时被替换成具体类型

fn largest<T>(list:&[T]) -> T {}

struct里的泛型类型参数可以和方法的泛型类型参数不同

struct Point<T, U> {
    x: T,
    y: U,
}

impl<T, U> Point<T, U> {
    fn mixup<V, W>(self, other: Point<V, W>) -> Point<T, W> {
        Point {
            x: self.x,
            y: other.y,
        }
    }
}

fn main() {
    let p1 = Point{x:5,y:4};
    let p2 = Point{x:"hello", y:'c'};
    let p3 = p1.mixup(p2);

    println!("p3.x={},p3.y={}",p3.x, p3.y);
}

10.2 Trait

trait:抽象的定义共享行为

trait bounds:泛型类型参数指定为实现了特定行为的类型

trait和接口有点类似

定义trait

把方法签名放在一起，来定义实现某种目的所必需的一组行为

  • 关键字trait
  • 只有方法签名，没有具体实现
  • trait可以有多个方法
  • 实现该trait的类型必须提供具体的方法实现

在类型上实现trait

impl xxxx for Tweet {}

pub trait Summary {
    fn summarize(&self) -> String;
}

pub struct NewsArticle {
    pub headline: String,
    pub location: String,
    pub author: String,
    pub content: String,
}

impl Summary for NewsArticle {
    fn summarize(&self) -> String {
        format!("{}, by {} ({})", self.headline, self.author, self.location)
    }
}

默认实现

可以在trait上添加默认的实现

默认实现的方法可以调用trait中的其他方法，即使这些方法没有默认实现

pub trait Summary {
    fn summarize_author(&self) -> String;

    fn summarize(&self) -> String {
        format!("(Read more from {} ...)", self.summarize_author())
    }
}

trait作为参数

  • impl Trait语法：适用于简单情况
  • Trait bound语法：可用于复杂情况
  • 使用***+***指定多个trait bound

trait作为返回值

  • impl trait：只能返回确定的同一种类型
  • 使用trait bound可以有条件的为实现了特定trait的类型实现方法
  • 为实现其他trait的任意类型有条件的实现某个trait

10.3 生命周期

  • rust每个引用都有自己的生命周期
  • 生命周期：引用保持有效的作用域
  • 大多数情况：生命周期是隐式的/可被推断的
  • 当引用的生命周期可能以不同的方式相互关联时：手动标注生命周期

函数的泛型生命周期

fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() {
        x
    } else {
        y
    }
}

生命周期的标注不会改变生命周期的长度。

生命周期的标注描述了多个引用的生命周期间的关系

泛型生命周期参数声明在函数名和参数列表之间。

生命周期’a的实际生命周期是：x和y两个生命周期中较小的那个

struct定义中的生命周期标注

struct ImportantExcerpt<'a> {
    part: &'a str,
}

生命周期的省略

  • 每个引用类型的参数都有自己的生命周期
  • 如果只有1个输入生命周期参数，那么该生命周期被赋予给所有的输出生命周期参数
  • 如果有多个输入生命周期参数，但其中一个是&self或&mut self（方法），那么self的生命周期会被赋给所有的输出生命周期参数
